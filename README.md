## 3. 디렉토리 별 프로그래밍 상세 설명

### 3-1. Final Project 


#### 3-1-1. 프로그래밍 기능 

    실시간 데이터 통신 구현: 다중 클라이언트가 서버에 접속하여 서로 메시지를 주고받을 수 있는 환경을 구축합니다. 

    멀티스레딩 활용: 여러 클라이언트의 요청을 동시에 처리할 수 있도록 스레드 기반 서버를 구현하여, 동시성 처리와 스레드 안전성(뮤텍스 사용)을 경험합니다. 

    에러 및 시그널 처리 학습: 서버의 비정상 종료나 예기치 않은 상황에서 안정성을 유지하기 위해 시그널 핸들링을 도입하여 안전한 네트워크 종료를 학습합니다. 


-------------------------------------------------------------


#### 3-1-2. 개발 내용 

**서버측 기능 :**  

    1. 클라이언트 연결 관리 

    서버는 여러 클라이언트의 접속을 동시에 허용하며, 최대 동시 접속 수는 상수 CLNT_MAX로 제한됩니다. 

    클라이언트가 연결되면, 각 클라이언트마다 새로운 스레드를 생성해 통신을 처리합니다. 

    2 .메시지 브로드캐스팅 

    특정 클라이언트가 메시지를 보내면, 해당 클라이언트를 제외한 다른 모든 클라이언트에게 메시지를 전달합니다. 

    send_all_clnt() 함수가 브로드캐스팅 역할을 수행합니다. 

    3. 뮤텍스 동기화 

    다중 클라이언트의 동시 접속 및 데이터 전송을 안전하게 처리하기 위해 뮤텍스를 사용하여 g_clnt_socks 배열의 접근을 보호합니다. 

    4. 비정상 종료 방지 (시그널 핸들링) 

    Ctrl + C, Ctrl + Z, Ctrl + \ 등으로 서버가 강제 종료되는 것을 방지하기 위해 시그널 핸들링을 적용했습니다. 

    강제 종료 시도를 감지하고, “서버를 종료하지 마세요”라는 메시지를 출력하며 종료를 방지합니다. 

**클라이언트측 기능 :** 

    1. 서버와의 연결 

    클라이언트는 서버에 연결하여 실시간으로 메시지를 주고받습니다. 

    2. 메시지 전송 

    클라이언트가 입력한 메시지를 서버로 전송합니다. 

    서버는 받은 메시지를 다른 클라이언트들에게 브로드캐스팅합니다. 

    3. 메시지 수신 

    다른 클라이언트로부터 전달된 메시지를 실시간으로 출력합니다.
